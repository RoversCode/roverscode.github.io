深度可分卷积

Posted on 2022-06-04 Edited on 2022-07-02 In Computer Vision , Convolutional Networks Views: Disqus: Symbols count in article: 1.5k Reading time ≈ 1 mins.

什么是深度可分卷积

现在来看深度可分卷积，这在深度学习领域要常用得多（比如 MobileNet 和 Xception）。深度可分卷积包含两个步骤：深度卷积核 1×1 卷积。

在描述这些步骤之前，有必要回顾一下我们之前介绍的 2D 卷积核 1×11×11×1 卷积。首先快速回顾标准的 2D 卷积。举一个具体例子，假设输入层的大小是 7×7×37×7×37×7×3（高×宽×通道），而过滤器的大小是 3×3×33×3×33×3×3。经过与一个过滤器的 2D 卷积之后，输出层的大小是 5×5×15×5×15×5×1（仅有一个通道）。

一般来说，两个神经网络层之间会应用多个过滤器。假设我们这里有 128 个过滤器。在应用了这 128 个 2D 卷积之后，我们有 128 个 5×5×15×5×15×5×1 的输出映射图（map）。然后我们将这些映射图堆叠成大小为 5×5×1285×5×1285×5×128 的单层。通过这种操作，我们可将输入层（7×7×3）（7×7×3）（7×7×3）转换成输出层（5×5×128）（5×5×128）（5×5×128）。空间维度（即高度和宽度）会变小，而深度会增大。

现在使用深度可分卷积，看看我们如何实现同样的变换。

首先，我们将深度卷积应用于输入层。但我们不使用 2D 卷积中大小为 3×3×33×3×33×3×3 的单个过滤器，而是分开使用 3 个核。每个过滤器的大小为 3×3×13×3×13×3×1。每个核与输入层的一个通道卷积（仅一个通道，而非所有通道！）。每个这样的卷积都能提供大小为 5×5×15×5×15×5×1 的映射图。然后我们将这些映射图堆叠在一起，创建一个 5×5×35×5×35×5×3 的图像。经过这个操作之后，我们得到大小为 5×5×3 的输出。现在我们可以降低空间维度了，但深度还是和之前一样。

第二步：为了扩展深度，我们应用一个核大小为 1×1×31×1×31×1×3 的 1×11×11×1 卷积。将 5×5×35×5×35×5×3 的输入图像与每个 1×1×31×1×31×1×3 的核卷积，可得到大小为 5×5×15×5×15×5×1 的映射图。

因此，在应用了 128 个 1×11×11×1 卷积之后，我们得到大小为 5×5×1285×5×1285×5×128 的层。

通过这两个步骤，深度可分卷积也会将输入层（7×7×3）（7×7×3）（7×7×3）变换到输出层（5×5×128）（5×5×128）（5×5×128）。

深度可分卷积的优势

所以，深度可分卷积有何优势呢？效率！相比于 2D 卷积，深度可分卷积所需的操作要少得多。

普通2D卷积的计算次数：

  • 回忆一下我们的 2D 卷积例子的计算成本。有 128 个 3×3×33×3×33×3×3 个核移动了 5×55×55×5 次，也就是 128∗3∗3∗3∗5∗5=86400128 * 3 * 3 * 3 * 5 * 5 =86400128∗3∗3∗3∗5∗5=86400 次乘法。

深度可分卷积的计算次数：

  • 在第一个深度卷积步骤，有 3 个 3×3×13×3×13×3×1 核移动 5×55×55×5 次，也就是 3∗3∗3∗1∗5∗5=6753*3*3*1*5*5 = 6753∗3∗3∗1∗5∗5=675 次乘法。
  • 在 1×11×11×1 卷积的第二步，有 128 个 1×1×31×1×31×1×3 核移动 5×55×55×5 次，即 128∗1∗1∗3∗5∗5=9600128 * 1 * 1 * 3 * 5 * 5 = 9600128∗1∗1∗3∗5∗5=9600 次乘法。
  • 因此，深度可分卷积共有 675+9600=10275675 + 9600 = 10275675+9600=10275 次乘法。这样的成本大概仅有 2D 卷积的 12%！
