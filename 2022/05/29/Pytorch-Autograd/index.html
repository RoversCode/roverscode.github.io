    Automatic defferentiation with TORCH.AUTOGRAD(torch.autograd的自动微分)

    When training neural networks, the most frequently used algorithm is back propagation.

    在该算法中，根据损失函数的梯度来调整参数(model weights)。

    To compute those gradients, PyTorch has a built-in differentiation engine(微分引擎) called torch.autograd.

    It supports automatic computation of gradient for any computational graph(计算图).

    这里有两个场景是你可以能想要禁用梯度追踪的:

    • 将神经网络中的一些参数标记为冻结参数(不参与梯度计算)。 This is a very common scenario for finetuning a pretrained network
    • To speed up computations when you are only doing forward pass, because computations on tensors that do not track gradients would be more efficient.

    More on Computational Graphs

    从概念上，autograd保存数据的记录(Tensor)和所有在由Function对象组成的directed acyclic graph(有向无环图 DAG)上执行的操作(新生成的Tensor也算)。In this DAG, 叶节点是输入Tensors，根节点是输出Tensors。从根节点到叶节点tracing这个图，你就可以使用链式法则自动计算梯度。

    In a forward pass, autograd 同时做了两件事情**：**

    • run the requested operation to compute a resulting tensor(运行请求的操作来计算结果张量)
    • maintain the operation’s gradient function in the DAG.(保留在DAG中operation的梯度函数)

    在DAG的root上调用**.backward()时，反向传播开始。然后**

    • computes the gradients from each .grad_fn,
    • 将这些梯度积累在tensor的grad属性上
    • 使用链式法则，传播到所有路径上的叶结点。
    💡 **DAGs are dynamic in PyTorch：**.一个很重要的事情要注意的是graph总是从零创造的；每一次调用`.backward()`，autograd都会填充新的graph。这也是为什么你可以在你的模型中使用控制流语句的原因；如果你需要，你可以在每一次的迭代中改变shape,size and operations。
